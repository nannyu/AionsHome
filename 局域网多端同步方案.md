# 局域网多端同步方案 —— 电脑做服务器，手机无缝连接

> 从 Aion Chat 项目中提取的完整实现方案。  
> 技术栈：Python FastAPI + WebSocket + 原生 JS，无任何前端框架依赖。

---

## 一、整体架构

```
┌──────────────┐          WiFi 局域网          ┌──────────────┐
│   电脑(服务端) │◄══════════════════════════════►│   手机(浏览器) │
│  FastAPI      │    HTTP REST + WebSocket      │              │
│  0.0.0.0:8080 │◄══════════════════════════════►│  另一台电脑   │
└──────────────┘                                └──────────────┘
```

**核心思路：**
- 电脑运行 FastAPI 服务，监听 `0.0.0.0:8080`（所有网卡）
- 手机通过浏览器访问 `http://电脑WiFi-IP:8080`
- 数据通过 REST API 进行 CRUD
- 实时同步通过 WebSocket 广播实现——任何一端的操作，其他端立即收到通知并更新 UI

## 二、服务端关键配置

### 2.1 监听所有网卡（0.0.0.0）

这是让手机能访问的**最关键一步**。`localhost` 或 `127.0.0.1` 只能本机访问，必须用 `0.0.0.0`：

```python
# main.py
if __name__ == "__main__":
    import uvicorn
    uvicorn.run("main:app", host="0.0.0.0", port=8080, reload=True)
```

### 2.2 Windows 防火墙放行

Windows 默认会拦截外部对本机端口的访问，需要添加入站规则：

```powershell
# 管理员权限执行
netsh advfirewall firewall add rule name="MyApp 8080" dir=in action=allow protocol=TCP localport=8080
```

或者用 Python 脚本自动提权执行：
```python
import subprocess
subprocess.run([
    "powershell", "-Verb", "RunAs", "-ArgumentList",
    "netsh advfirewall firewall add rule name='MyApp 8080' dir=in action=allow protocol=TCP localport=8080"
])
```

### 2.3 查看电脑 WiFi IP

手机需要知道电脑的局域网 IP 才能连接：

```powershell
ipconfig
# 找到 "无线局域网适配器 WLAN" 下的 IPv4 地址，通常是 192.168.x.x
```

**⚠️ 注意事项：**
- `169.254.x.x` 是无效地址（APIPA），说明网络未正确连接
- VPN 软件（如 singbox）可能干扰局域网路由，必要时关闭或加直连规则
- 电脑和手机必须在**同一个 WiFi** 下

## 三、WebSocket 实时同步（后端）

### 3.1 连接管理器

管理所有 WebSocket 连接，支持广播消息给所有客户端：

```python
import json
from fastapi import FastAPI, WebSocket, WebSocketDisconnect

class ConnectionManager:
    def __init__(self):
        self.active: list[WebSocket] = []

    async def connect(self, ws: WebSocket):
        """接受新的 WebSocket 连接"""
        await ws.accept()
        self.active.append(ws)

    def disconnect(self, ws: WebSocket):
        """移除断开的连接"""
        self.active.remove(ws)

    async def broadcast(self, data: dict, exclude: WebSocket = None):
        """
        广播消息给所有已连接客户端。
        exclude 参数可排除消息发送者自己（避免重复）。
        """
        msg = json.dumps(data, ensure_ascii=False)
        for ws in self.active.copy():  # 用 copy() 避免迭代时修改列表
            if ws is not exclude:
                try:
                    await ws.send_text(msg)
                except:
                    self.active.remove(ws)  # 自动清理已断开的连接

manager = ConnectionManager()
```

### 3.2 WebSocket 端点

```python
@app.websocket("/ws")
async def websocket_endpoint(ws: WebSocket):
    await manager.connect(ws)
    try:
        while True:
            await ws.receive_text()  # 保持连接，客户端可发心跳
    except WebSocketDisconnect:
        manager.disconnect(ws)
```

### 3.3 在 API 操作后广播通知

每次数据变更（增删改），调用 `broadcast` 告知所有客户端：

```python
@app.post("/api/items")
async def create_item(body: ItemCreate):
    # 1. 写入数据库
    item = save_to_db(body)
    
    # 2. 广播给所有端 → 手机/PC 都会收到
    await manager.broadcast({
        "type": "item_created",   # 事件类型（前端用来区分处理逻辑）
        "data": item              # 数据载荷
    })
    
    return item

@app.put("/api/items/{id}")
async def update_item(id: str, body: ItemUpdate):
    item = update_in_db(id, body)
    await manager.broadcast({"type": "item_updated", "data": item})
    return item

@app.delete("/api/items/{id}")
async def delete_item(id: str):
    delete_from_db(id)
    await manager.broadcast({"type": "item_deleted", "data": {"id": id}})
    return {"ok": True}
```

**消息协议设计（自定义 JSON）：**
```json
{
  "type": "item_created",     // 事件类型字符串
  "data": { ... }             // 具体数据，结构取决于业务
}
```

常用事件类型示例：
| type | 触发时机 | data 内容 |
|------|---------|-----------|
| `conv_created` | 新建对话 | 完整对话对象 |
| `conv_updated` | 修改标题/配置 | 变更的字段 |
| `conv_deleted` | 删除对话 | `{ id }` |
| `msg_created` | 新消息 | 完整消息对象 |
| `msg_updated` | 编辑消息 | 完整消息对象 |
| `msg_deleted` | 删除消息 | `{ id, conv_id }` |

## 四、WebSocket 实时同步（前端）

### 4.1 建立连接 + 自动重连

```javascript
let ws = null;

function connectWS() {
  // 自动适配 http/https → ws/wss
  const proto = location.protocol === "https:" ? "wss:" : "ws:";
  ws = new WebSocket(`${proto}//${location.host}/ws`);

  // 收到消息 → 解析 JSON → 分发处理
  ws.onmessage = e => handleSync(JSON.parse(e.data));

  // 断线自动重连（2秒后重试）
  ws.onclose = () => setTimeout(connectWS, 2000);
}
```

**关键点：**
- 用 `location.host` 自动获取当前服务地址（PC 是 `localhost:8080`，手机是 `192.168.x.x:8080`），无需硬编码
- `onclose` 自动重连，网络抖动或手机息屏后恢复都能自动恢复同步
- 不需要手动写心跳，浏览器的 WebSocket 实现自带 ping/pong

### 4.2 消息分发处理

根据 `type` 字段区分事件，更新本地数据 + 重新渲染 UI：

```javascript
function handleSync(msg) {
  const { type, data } = msg;

  if (type === "item_created") {
    // 检查本地是否已存在（避免自己发的消息重复）
    if (!localItems.find(i => i.id === data.id)) {
      localItems.push(data);
      renderList();          // 重新渲染 UI
    }

  } else if (type === "item_updated") {
    const idx = localItems.findIndex(i => i.id === data.id);
    if (idx >= 0) {
      localItems[idx] = data;  // 用服务端数据覆盖本地
      renderList();
    }

  } else if (type === "item_deleted") {
    localItems = localItems.filter(i => i.id !== data.id);
    renderList();
  }
}
```

### 4.3 防重复的技巧

当用户在 A 端操作（如发消息），A 端会：
1. 先在本地创建一条临时数据（带临时 ID 如 `"temp_user"`），立即显示在 UI 上
2. 调 REST API 保存到数据库
3. 后端保存后 broadcast → A 端也会收到这条广播
4. A 端收到后，用真实数据替换临时数据

```javascript
// 发送时：先显示临时消息
localMessages.push({ id: "temp_user", content: text, role: "user" });
renderMessages();

// 调 API
await fetch("/api/messages", { method: "POST", body: JSON.stringify({ content: text }) });

// handleSync 中处理：
if (type === "msg_created") {
  // 如果本地有临时消息，替换为真实消息
  const tempIdx = localMessages.findIndex(m => m.id === "temp_user");
  if (tempIdx >= 0 && data.role === "user") {
    localMessages[tempIdx] = data;  // 替换
  }
  // 否则是其他端发来的新消息
  else if (!localMessages.find(m => m.id === data.id)) {
    localMessages.push(data);
  }
  renderMessages();
}
```

## 七、完整最小示例

把上面所有内容整合，以下是一个可以直接跑的最小 demo：

### 7.1 后端 `server.py`

```python
"""最小局域网同步 demo"""
import json, time
from fastapi import FastAPI, WebSocket, WebSocketDisconnect
from fastapi.responses import HTMLResponse
from contextlib import asynccontextmanager

# ── 内存存储（实际项目换成数据库）──
items = []

# ── WebSocket 管理 ──
class ConnectionManager:
    def __init__(self):
        self.active: list[WebSocket] = []

    async def connect(self, ws: WebSocket):
        await ws.accept()
        self.active.append(ws)

    def disconnect(self, ws: WebSocket):
        self.active.remove(ws)

    async def broadcast(self, data: dict):
        msg = json.dumps(data, ensure_ascii=False)
        for ws in self.active.copy():
            try:
                await ws.send_text(msg)
            except:
                self.active.remove(ws)

manager = ConnectionManager()
app = FastAPI()

# ── REST API ──
@app.get("/api/items")
async def list_items():
    return items

@app.post("/api/items")
async def add_item(body: dict):
    item = {"id": str(int(time.time() * 1000)), "text": body["text"]}
    items.append(item)
    await manager.broadcast({"type": "item_added", "data": item})
    return item

# ── WebSocket ──
@app.websocket("/ws")
async def ws_endpoint(ws: WebSocket):
    await manager.connect(ws)
    try:
        while True:
            await ws.receive_text()
    except WebSocketDisconnect:
        manager.disconnect(ws)

# ── 前端页面 ──
@app.get("/")
async def index():
    return HTMLResponse("""
    <!DOCTYPE html>
    <html>
    <head><meta charset="UTF-8"><meta name="viewport" content="width=device-width,initial-scale=1.0"></head>
    <body style="font-family:sans-serif;max-width:400px;margin:40px auto;background:#111;color:#eee;padding:20px">
      <h2>局域网同步 Demo</h2>
      <div id="list"></div>
      <input id="inp" placeholder="输入内容..." style="width:70%;padding:8px;background:#222;color:#eee;border:1px solid #444;border-radius:6px">
      <button onclick="add()" style="padding:8px 16px;background:#5b9bd5;color:#fff;border:none;border-radius:6px;cursor:pointer">添加</button>
      <script>
        const list = document.getElementById('list');
        const inp = document.getElementById('inp');
        let items = [];

        // 加载已有数据
        fetch('/api/items').then(r=>r.json()).then(d => { items = d; render(); });

        // WebSocket 实时同步
        function connectWS() {
          const ws = new WebSocket((location.protocol==='https:'?'wss:':'ws:')+'//'+location.host+'/ws');
          ws.onmessage = e => {
            const msg = JSON.parse(e.data);
            if (msg.type === 'item_added' && !items.find(i=>i.id===msg.data.id)) {
              items.push(msg.data);
              render();
            }
          };
          ws.onclose = () => setTimeout(connectWS, 2000);
        }
        connectWS();

        function render() {
          list.innerHTML = items.map(i => '<p>• '+i.text+'</p>').join('');
        }

        function add() {
          if (!inp.value.trim()) return;
          fetch('/api/items', {method:'POST', headers:{'Content-Type':'application/json'}, body: JSON.stringify({text:inp.value})});
          inp.value = '';
        }
      </script>
    </body>
    </html>
    """)

if __name__ == "__main__":
    import uvicorn
    uvicorn.run("server:app", host="0.0.0.0", port=8080, reload=True)
```

**运行：**
```bash
pip install fastapi uvicorn
python server.py
```

然后电脑打开 `http://localhost:8080`，手机打开 `http://电脑IP:8080`，两边输入数据会实时同步。

## 八、排查清单

| 问题 | 排查方法 |
|------|---------|
| 手机打不开页面 | 确认同一 WiFi；用 `ipconfig` 查 IP；试 `ping 电脑IP` |
| 连接被拒绝 | 检查 Windows 防火墙是否放行了端口 |
| VPN 干扰 | 关闭 VPN 或添加局域网直连规则 |
| WebSocket 断线 | 前端 `onclose` 会自动重连，正常现象 |
| 多端数据不一致 | 检查 broadcast 是否覆盖了所有 CRUD 操作 |
| 手机息屏后不同步 | 息屏时 WebSocket 会断，亮屏后 `onclose→setTimeout→connectWS` 自动恢复 |

## 九、非局域网连接：使用 Tailscale 远程访问

> 上面所有方案都基于**同一局域网（WiFi）**，如果你出门在外（手机用流量/外面的 WiFi），是无法直接访问家里电脑的。
> **Tailscale** 可以在你的设备之间建立一个虚拟局域网（WireGuard VPN），实现"在任何网络环境下像局域网一样互访"。

### 9.1 什么是 Tailscale？

Tailscale 是一个零配置的组网工具：
- **免费**（个人用户 20 台设备额度够用）
- **不需要公网 IP**，不需要内网穿透
- **不需要自建服务器**，P2P 直连（延迟很低）
- 每台设备会分配一个固定的 `100.x.x.x` 虚拟 IP

### 9.2 下载和安装

| 平台 | 下载地址 |
|------|---------|
| Windows | https://tailscale.com/download/windows |
| Android | Google Play 搜索「Tailscale」或从官网下载 APK |
| iOS | App Store 搜索「Tailscale」 |
| macOS | https://tailscale.com/download/mac |
| Linux | https://tailscale.com/download/linux |

> 官网：https://tailscale.com/download

### 9.3 配置步骤

1. **电脑（服务端）和手机都安装 Tailscale**
2. **两台设备都用同一个账号登录**（支持 Google/Microsoft/GitHub 账号）
3. 登录后，Tailscale 会自动分配 IP 地址：
   - 电脑端：打开 Tailscale → 右下角托盘图标 → 鼠标悬浮看到类似 `100.xx.xxx.xx` 的地址
   - 也可以打开 https://login.tailscale.com/admin/machines 查看所有设备的 IP
4. **手机浏览器打开 `http://100.xx.xxx.xx:8080`**（替换为你电脑的 Tailscale IP）即可远程访问

### 9.4 使用场景

```
场景一：在家（同一 WiFi）
  手机 → http://192.168.x.x:8080  （局域网 IP，速度最快）

场景二：出门在外（手机用流量）
  手机 → http://100.xx.xxx.xx:8080 （Tailscale 虚拟 IP，需要两端都开着 Tailscale）
```

### 9.5 注意事项

- 电脑端 Tailscale **必须保持运行**（右下角托盘图标显示已连接）
- Aion Chat 服务也必须在跑，出门前确认 `一键启动.bat` 窗口没关
- 手机端 Tailscale 也要保持**已连接**状态（通知栏有 VPN 图标）
- 首次使用可能需要科学上网才能登录 Tailscale 账号，登录成功后正常使用不需要
- Tailscale IP 是固定的，不会变，记住一次就行
